## 14. Línea gráfica (sistema visual canónico)

> **Fuente de verdad:** este sistema visual está implementado hoy en `index.html`. Toda iteración nueva debe respetar estos tokens y patrones salvo decisión explícita de rebranding.

### 14.1 Paleta — tokens exactos

Configurar en `tailwind.config` inline dentro del `<head>`, bajo `theme.extend.colors`:

| Token       | Hex         | Uso canónico                                                               |
|-------------|-------------|----------------------------------------------------------------------------|
| `ink`       | `#0A0E1A`   | Fondo de secciones dark · color de texto sobre fondos claros · CTA dark.   |
| `ink2`      | `#111726`   | Superficie secundaria sobre ink (variantes, cards hundidas).               |
| `ivory`     | `#F7F6F2`   | Fondo principal de secciones light · texto sobre fondos dark.              |
| `paper`     | `#FBFAF7`   | Superficie secundaria sobre ivory (cards, quotes).                         |
| `line`      | `#1C2333`   | Líneas y bordes sutiles en fondos dark cuando se requiere más contraste.   |
| `accent`    | `#815BD4`   | **Color de marca principal** — violeta. CTA primario, "Lab" del logo, acentos, highlights de texto, bordes activos. |
| `accent2`   | `#A78BFA`   | **Violeta claro** — halos, glows, nodos de animación, degradados de acento.|
| `muted`     | `#64748B`   | Texto secundario neutral sobre fondos claros (subtítulos, eyebrows).        |

> **Importante:** la paleta de marca es **violeta (#815BD4 / #A78BFA)**, no emerald. Esta decisión ya está tomada y debe respetarse en todo asset nuevo (OG image, iconos, ilustraciones, decks). El verde emerald queda descartado.

**Reglas de uso:**
- **Fondos dark** usan `ink` como base y `white/[.02]` – `white/[.10]` para superficies sobrepuestas (bg + border).
- **Fondos light** usan `ivory` como base y `paper` + `border-ink/10` para cards.
- **El accent violeta nunca se usa como color de fondo de sección.** Siempre es acento puntual: bullets, subrayados, bordes activos, halos difuminados (`bg-accent/10..20 blur-3xl`), texto destacado.
- **Jerarquía de énfasis de texto sobre dark:** `text-white` (100%) → `text-white/85` → `text-white/70` → `text-white/50` → `text-white/40`.
- **Jerarquía de énfasis de texto sobre light:** `text-ink` → `text-ink/80` → `text-ink/70` → `text-muted`.

### 14.2 Tipografía

**Cargar vía Google Fonts** (`<link>` con `preconnect`):
```
Inter:   300, 400, 500, 600, 700, 800
Fraunces (opsz 9..144): 400, 500, 600
display=swap
```

| Familia     | Token Tailwind | Uso                                                                |
|-------------|----------------|---------------------------------------------------------------------|
| Inter       | `font-sans`    | Cuerpo, UI, eyebrows, labels, botones. **Default del `<body>`**.   |
| Fraunces    | `font-display` | Titulares (h1–h3), quotes destacadas, números grandes de stats.    |

**Escalas (obligatorias en hero y secciones principales):**
- **H1 (hero):** `font-display text-[44px] leading-[1.03] md:text-[76px] md:leading-[.98] tracking-tightest`, con `.grad-text` aplicado.
- **H2 de sección:** `font-display text-4xl md:text-6xl tracking-tightest leading-[1.02]`. Patrón: primera frase full color + segunda frase en `text-muted` (light) o `text-white/50` (dark), separadas por `<span>`.
- **H3 de cards:** `font-display text-2xl leading-tight`.
- **Body largo:** `text-lg md:text-xl text-white/70 leading-relaxed` (hero) · `text-lg text-ink/70` (secciones light).
- **Eyebrow de sección:** `text-xs uppercase tracking-[0.22em] text-muted` (light) o `text-white/50` (dark). **Formato obligatorio:** `"0N · Nombre de sección"` (ver 14.9).
- **Labels de formulario / microcopy:** `text-xs uppercase tracking-wider text-white/60` o `text-ink/60`.

**Letter-spacing custom:** extender `letterSpacing.tightest = '-0.035em'` y aplicarlo a todo titular display. Habilitar features OpenType en body:
```css
body { font-feature-settings: "ss01", "cv11"; }
```

### 14.8 Motion

**Librerías: ninguna.** Todo motion va en CSS + un único IntersectionObserver.

- **Reveal on scroll:** clase `.reveal` (opacity 0, translateY 18px) → `.visible` (IntersectionObserver con `threshold: 0.12`, `unobserve` al entrar). Transición `.8s cubic-bezier(.2,.6,.2,1)`.
- **Hover lift en cards:** `translateY(-4px)` + border accent.
- **Hover lift en botones:** `translateY(-1px)` + shadow violeta.
- **Smooth scroll** global (`html { scroll-behavior: smooth; }`).
- **Pulse point del eyebrow "live":** `w-1.5 h-1.5 rounded-full bg-accent animate-pulse`.
- **Ilustración planeta (hero):** SVG 400×400 con múltiples `<g>` rotando a distintas velocidades (`spin-cw` 22s/40s/60s, `spin-ccw` 28s), core pulse (`r: 5→9→5` en 2.2s), nodos con `<animate attributeName="opacity">` 2.2–3.6s, conexiones con dash-array animado 4–6.6s. **Respetar `prefers-reduced-motion: reduce` deshabilitando todas estas animaciones.**
- **Marquee:** `scroll-x` 40s linear infinite, track duplicado para loop sin salto.
